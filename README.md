# 数据防泄漏 (DLP) 安全审计终端

> **Data Loss Prevention Audit Terminal**

本项目是一款基于多模态检索技术的数据防泄漏审计工具。旨在辅助安全管理员及合规审查人员，通过自动化特征匹配与视觉识别技术，快速定位分布于本地文件系统、关系型数据库、Web 端以及图像介质中的潜在敏感信息，并生成结构化的审计报告。

## 核心功能模块 (Features)

系统底层采用高度解耦的模块化设计，集成四大核心审查引擎：

1. **本地文件深度检查 (Local File Deep Scanner)**
   * **多格式解析**：支持对 `.txt`, `.docx`, `.pdf` 等常见文档格式的文本提取与编码自适应解析。
   * **隐藏属性检测**：通过调用操作系统底层 API (`ctypes`)，实现对系统级隐藏文件及特殊权限目录的遍历与审查。
   * **加密状态识别**：自动捕获并标记设有密码保护或权限限制的加密文档，防止审计过程发生遗漏。

2. **数据库安全扫描 (Database Scanner)**
   * **直连取证**：支持基于标准凭证直连目标数据库（如 MySQL），实现静态数据流的抽取与合规审计。
   * **异常管理**：内置数据库连接池管理与异常重试机制，保障大批量数据检索时的程序稳定性。

3. **网页动态爬虫 (Web Crawler)**
   * **广度优先遍历 (BFS)**：采用可控深度的图遍历算法（默认深度设置 Depth=2），对目标站点进行页面拓扑分析与敏感词捕获。
   * **重定向防护**：内置已访问路径哈希池，有效避免程序陷入动态网页的无限重定向循环。

4. **图像 OCR 识别 (Image OCR Scanner)**
   * **光学字符识别**：离线集成 `PaddleOCR` 轻量级神经网络模型，实现图像内文本的提取与审查。
   * **多环境兼容**：配置为 CPU 推理模式，解除对特定 GPU 硬件及 CUDA 驱动的依赖，提升跨设备部署的可靠性与可用性。

## 系统架构与技术栈 (Architecture & Tech Stack)

本项目采用松耦合的 MVC 衍生架构，实现业务逻辑、表现层与数据持久层的严格分离。

* **编程语言**: Python 3.10+
* **前端展现**: 基于 `CustomTkinter` 构建的异步非阻塞 GUI，包含实时日志监控终端。
* **数据处理**: `Pandas` (用于内存数据清洗与聚合)
* **视觉模型**: `PaddlePaddle` & `PaddleOCR` (CPU Vision)
* **并发控制**: 基于 `threading` 的多线程架构，确保底层 I/O 密集型扫描与 UI 渲染解耦。
* **编译部署**: `PyInstaller` (独立二进制构建)

## 部署与使用 (Deployment & Usage)

### 选项 A：开箱即用（推荐独立部署方案）
对于无 Python 环境的终端设备，可直接使用已编译的独立可执行程序。
1. 获取编译后目录并进入 `dist/DLP_DeepWatcher/`。
2. 双击运行 `DLP_DeepWatcher.exe` 启动主程序。
3. 在系统左侧导航栏选择对应的审计模式，输入目标路径或连接参数。
4. 点击启动扫描，等待进程结束后，可通过界面右侧按钮导出结构化报告。

### 选项 B：基于源码运行（二次开发方案）
如需进行二次开发或规则库扩充：
1. 克隆项目并进入根目录。
2. 激活 Python 虚拟环境，安装相关依赖项：
   ```bash
   pip install -r requirements.txt
   ```
3. 执行启动脚本：
   ```bash
   python run.py
   ```
## 审计报告输出 (Reporting)

系统采用统一的 `ScanSummary` 数据模型进行结果聚合。导出的 Excel 报告包含两个工作表：

* **Dashboard (概览视图)**：包含扫描任务名称、扫描深度、耗时、总检阅文件数及敏感特征条目总计等宏观统计数据。
* **Details (明细视图)**：包含具体的风险源路径、代码/文本行号、触发特征词及上下文快照（Context），为后续的合规追责与人工复核提供取证凭据。
